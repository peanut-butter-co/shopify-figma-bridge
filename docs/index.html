How It Works

Three phases. One command to run them all.

1

Analyze

Point at any Shopify theme. Tokens extracted automatically.

2

Build

Variables, text styles, components created in Figma.

3

Design

Compose pages from real components. Desktop + mobile.

Features

Everything you need to bridge the design-development gap.

Any Shopify Theme

Works with Horizon, Dawn, Refresh, or any custom theme. Theme profiles accelerate known themes.

Token Extraction

Colors, typography, spacing, radii -- extracted from settings_schema.json and settings_data.json automatically.

Variable Architecture

Primitives to Tokens with color scheme modes. Changes cascade everywhere.

Component Library

Atoms, blocks, sections -- all as Figma components with proper auto-layout and variable bindings.

Page Composition

14 template types supported. Desktop + mobile paired. Built from real component instances.

Instance Enforcement

Every sub-element is a component instance. Built-in audit catches violations.

Shopify Validation

Schema-aware JSON validation catches range errors, dependency issues, and block type mismatches.

Brand Onboarding

Brand guidelines to Primitives to all templates. New client setup in minutes, not days.

Bidirectional Sync

Figma to Shopify and back. Design changes update template JSON. Code changes update Figma.

Self-Learning

Gotchas files per skill. The system gets smarter with every build it completes.

Quick Start

Up and running in three steps.

# Install
curl -sfL https://raw.githubusercontent.com/peanut-butter-co/shopify-figma-bridge/main/install.sh | bash

# Configure
claude
> /setup

# Build everything
> /build-design-system
